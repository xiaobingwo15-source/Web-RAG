## Deployment Steps

### Step 1: Backend → Render

**What to do:**
1. Push code to GitHub (if not already)
2. Go to Render Dashboard → New → Web Service
3. Connect your GitHub repo
4. Configure:
   - **Build Command:** `cd backend && pip install -r requirements.txt`
   - **Start Command:** `cd backend && uvicorn app.main:app --host 0.0.0.0 --port $PORT`
   - **Runtime:** Python 3
   - **Plan:** Free
5. Add all environment variables from the list above
6. Deploy

**What to notice:**
- Render free tier spins down after 15min idle → first request takes 30-60s
- Use UptimeRobot to keep alive during dev phase
- Free tier has 750 hours/month — enough for one always-on service
- Render assigns a random URL like `https://your-app.onrender.com`
- Custom domain available on paid tier only

**Health endpoint needed:**
```python
# Add to backend/app/main.py
@app.get("/health")
async def health():
    return {"status": "ok"}
```

### Step 2: Frontend → Vercel

**What to do:**
1. Go to Vercel Dashboard → Add New Project
2. Import your GitHub repo
3. Configure:
   - **Framework Preset:** Vite
   - **Root Directory:** `frontend`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. Add environment variable:
   - `VITE_API_URL` → your Render backend URL (e.g., `https://your-app.onrender.com`)
5. Deploy

**What to notice:**
- Vercel auto-deploys on every git push to main
- Free tier: 100GB bandwidth, serverless functions included
- Assigns URL like `https://your-app.vercel.app`
- Custom domain supported on free tier (bring your own domain)
- Update CORS in backend to allow Vercel domain

### Step 3: CORS Configuration

**Update backend to allow frontend domain:**
```python
# backend/app/main.py
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",                    # local dev
        "https://your-app.vercel.app",              # production frontend
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### Step 4: UptimeRobot Keep-Alive (Dev Phase Only)

**What to do:**
1. Create UptimeRobot account
2. Add Monitor:
   - **Type:** HTTP(s)
   - **URL:** `https://your-app.onrender.com/health`
   - **Interval:** 5 minutes
   - **Name:** Render Keep-Alive
API_key = u3540713-e9064ed06e633b97c88eea1b
3. Done — backend stays warm

**When to disable:**
- When you upgrade to Render paid tier ($7/mo)
- When you have paying tenants and need guaranteed uptime

### Step 5: Verify Deployment

**Checklist:**
```
□ Backend health endpoint responds: GET /health
□ Frontend loads in browser
□ Login/signup works
□ Chat sends message and receives response
□ Document upload works (admin)
□ RAG retrieval returns document-based answers
□ Langfuse traces appear in dashboard
□ Supabase RLS prevents cross-user data access
```

## Multi-Tenant Roadmap (Future — Phase 2)

> This is NOT needed for initial deployment. Do this when you have multiple websites wanting to use your RAG system.

### What Changes

```
CURRENT:
  1 admin → documents → RAG → clients

TARGET:
  N admins (1 per website) → each has own documents → RAG → their clients only
```

### Database Changes Needed

```sql
-- New table: organizations (one per website/admin)
CREATE TABLE organizations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  api_key TEXT UNIQUE NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Add org_id to existing tables
ALTER TABLE documents ADD COLUMN org_id UUID REFERENCES organizations(id);
ALTER TABLE chat_threads ADD COLUMN org_id UUID REFERENCES organizations(id);
ALTER TABLE document_chunks ADD COLUMN org_id UUID REFERENCES organizations(id);

-- RLS policies for org isolation
-- Users can only access their org's data
```

### Qdrant Changes

```python
# Each tenant gets own collection
collection_name = f"chunks_{org_slug}"
# e.g., chunks_acme_corp, chunks_globex_inc
```

### External API Design

```bash
# Third-party websites call your RAG API:
POST https://your-app.onrender.com/api/v1/chat
Headers:
  X-API-Key: org_acme_abc123
  Content-Type: application/json
Body:
  {
    "message": "What are your warranty terms?",
    "client_id": "their_user_456"
  }

# Response scoped to that org's documents only
```

## Risk Warnings

| Risk | Impact | Mitigation |
|------|--------|------------|
| Render free tier spins down | First request takes 30-60s | UptimeRobot keep-alive (dev phase) |
| Supabase free DB pauses after 7 days idle | First query after pause takes ~30s | Any request wakes it; UptimeRobot helps |
| UptimeRobot goes down | Render spins down | Upgrade to Render paid tier when revenue justifies |
| Qdrant free tier (1GB) fills up | Can't store more chunks | Monitor usage, upgrade when needed |
| Cross-tenant data leak | Critical security issue | RLS policies + separate Qdrant collections |
| OpenRouter runs out of credits | Chat stops working | Monitor balance, add auto-recharge |
