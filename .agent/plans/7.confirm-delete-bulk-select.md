# 7. Confirm Dialog + Bulk Select for Delete Operations

## Goal
Add proper confirmation dialogs for all delete operations and multi-select checkboxes for bulk document deletion.

## Scope
- **Frontend only** — no backend changes needed (single-delete endpoint is called per item)
- **Components affected**: 4 files

## Plan

### Step 1: Create `ConfirmDialog` component
**File**: `frontend/src/components/ConfirmDialog.tsx` (new)

A reusable modal dialog matching the existing `DocumentPreviewModal` overlay pattern:
- Props: `open`, `title`, `message`, `confirmLabel`, `danger` (boolean for red styling), `onConfirm`, `onCancel`
- Render: fixed overlay with `bg-black/60 backdrop-blur-sm`, centered card
- Keyboard: Escape to cancel, Enter to confirm
- Uses `lucide-react` icons (AlertTriangle for danger, Info for neutral)

### Step 2: Update `DocumentUpload.tsx` — multi-select + confirmation dialog
**File**: `frontend/src/components/DocumentUpload.tsx`

Changes:
- Add `selectedIds: Set<string>` state for multi-select
- Add checkbox on each document row (visible always when `onDelete` prop exists)
- Add "Select all" checkbox in the header row
- Add bulk delete toolbar: shows count + "Delete selected" button when selection is non-empty
- Replace double-click confirm with `ConfirmDialog` for both single and bulk delete
- Add `onDeleteMany?: (ids: string[]) => Promise<void>` prop for bulk delete
- Remove `confirmDeleteId` / `confirmTimerRef` state (no longer needed)

### Step 3: Update `useDocuments.ts` — add `deleteDocuments` bulk hook
**File**: `frontend/src/hooks/useDocuments.ts`

Add `deleteDocuments(ids: string[])` that:
- Calls `apiDeleteDocument` for each ID (sequentially to avoid rate limits)
- Removes all deleted IDs from local state in one `setDocuments` call
- Returns `{ deleted: number; errors: string[] }`

### Step 4: Wire bulk delete in `AdminPage.tsx`
**File**: `frontend/src/pages/AdminPage.tsx`

- Pass `deleteDocuments` as `onDeleteMany` to `DocumentUpload`

### Step 5: Add confirmation to thread delete — `ChatThreadList.tsx`
**File**: `frontend/src/components/ChatThreadList.tsx`

- Add `confirmDeleteId` state + `ConfirmDialog`
- Single-click Trash2 opens dialog, confirm executes delete

### Step 6: Add confirmation to thread delete — `ChatHistoryPanel.tsx`
**File**: `frontend/src/components/ChatHistoryPanel.tsx`

Same pattern as ChatThreadList.

## Verification
- Upload 3+ documents → checkboxes appear on each row
- Select all → deselect one → bulk delete shows correct count
- Click "Delete selected" → confirm dialog appears with filename list
- Confirm → all selected docs removed from list
- Single delete Trash2 → same confirm dialog
- Chat thread Trash2 → confirm dialog before delete
- Escape key / click outside → dismisses dialog
