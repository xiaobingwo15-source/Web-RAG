# Plan: Fix CoVe & HyDE Failure Handling

**Complexity:** ✅ Simple

## Problem

Two warnings appear in production logs when clients ask questions:
- `CoVe question generation failed: ` (empty error)
- `HyDE hypothetical answer generation failed: ` (empty error)

Both failures are silently swallowed with **fail-open** behavior:
- HyDE failure → pipeline continues with fewer chunks (acceptable, additive step)
- CoVe failure → returns `verified: True`, falsely logging "Chain-of-Verification passed"
- `_verify_single_question` failure → returns `supported: True`, hiding potential hallucinations

The empty error messages suggest the LLM returned empty/None responses, not exceptions.

## Changes

### File: `backend/app/services/agents/doc_rag_agent.py`

### 1. Add retry helper (insert after imports, ~line 14)

Add a simple `_retry_once` async helper that retries a coroutine once after a short delay. Keeps it minimal — no tenacity dependency, no exponential backoff. Just one retry for transient LLM failures.

```python
async def _retry_once(coro_factory, delay: float = 1.0):
    """Retry a coroutine once on failure. Returns result or raises."""
    try:
        return await coro_factory()
    except Exception:
        await asyncio.sleep(delay)
        return await coro_factory()
```

Change `generate_hypothetical_answer` to retry the LLM call once on failure. Also log the specific failure reason more clearly when the response is empty vs. an exception.

```python
        try:
            response = await asyncio.wait_for(
                client.chat.completions.create(...),

            answer = (response.choices[0].message.content or "").strip()
            if answer:
                logger.info(f"HyDE hypothetical answer generated ({len(answer)} chars)")
                return answer
            logger.warning(f"HyDE attempt {attempt+1}: empty response")

            logger.warning(f"HyDE attempt {attempt+1}: timeout (8s)")
        except Exception as e:
            logger.warning(f"HyDE attempt {attempt+1} failed: {e}")
        if attempt == 0:
            await asyncio.sleep(1.0)
    return None
```

## What NOT to change

- HyDE returning `None` on failure is fine — it's an additive enrichment step. The pipeline continues with RAG-Fusion chunks. No need to mark the answer as ungrounded just because HyDE didn't run.
- The groundedness retry loop (separate from CoVe) is unaffected.
- No new dependencies needed.
